nginx

nginx.conf是nginx的主要配置文件。

  1. 顶层下可以有events、http层。
  2. http层下允许有多个server层，用于对不同的网址做不同的配置。
  3. server层下允许有多个location层，用于对不同的(路由)地址进行不同的转发操作。

修改完nginx.conf配置文件后，可以使用如下命令对进行语法检查。

nginx -t -c /etc/nginx/nginx.conf

nginx模块

nginx编译选项 模块作用
ngx_http_core_module 包含一些核心的http的参数配置，对应nginx的配置区块部分
ngx_http_access_module 访问控制模块，用来控制网站用户对nginx的访问
ngx_http_gzip_module 压缩模块，对nginx返回的数据压缩，数据性能优化模块
ngx_http_fastcgi_module fastcgi模块，和动态应用相关的模块
ngx_http_upstream_module 负载均衡模块，可以实现网址的负载均衡功能以及节点节点健康检查
ngx_http_rewrite_module URL地址重写模块
ngx_http_proxy_module proxy代理模块
ngx_http_limit_conn_module 限制用户并发连接数以及请求数模块
ngx_http_limit_req_module 限制nginx request processing rate根据定义的key(限制请求，一个连接可以有多个请求)
ngx_http_log_module 访问日志模块，指定的格式记录nginx客户访问日志等信息
ngx_http_auth_basic_module web认证模块，这只web用户通过账号密码访问nginx
ngx_http_ssl_module ssl模块，用户加密的http连接，比如https

在nginx.conf中有三种常见变量：http请求变量、nginx内置变量、自定义变量。

一般可以通过在shell脚本中通过curl获取到这些监控信息后，再通过python将这些信息上报到监控平台组件(openfalcon)，达到可视化监控nginx连接情况的目的。

资源服务器

将nginx作为资源服务器，提供文件下载功能。

限流模块

  • 连接频率限制ngx_limit_conn_module，通过limit_req_zone用来限制单位时间内的请求数目，以及速度限制。
  • 请求频率限制ngx_limit_req_module，通过limit_req_conn用来限制同一时间连接数，即并发限制。

对于http协议的连接和请求：Http是建立在TCP，通过TCP三次握手，在连接的基础上进行Http请求，也就是说可能存在一个连接多次请求。

HTTP协议版本 连接关系
HTTP1.0 TCP不能复用
HTTP1.1 顺序性TCP复用
HTTP2.0 多路复用TCP

访问限制

简单实例

1.每个IP在每秒内只能访问2次。(nginx的时间处理单位是毫秒，也就是说会将每秒2次转换为每500毫秒1次)

2.添加请求缓存区，这时表示每个IP最多允许4个突发请求。假设某个IP在10ms内发送了6个请求，这时第一个请求被立即执行，第2~5个放入缓存区，第6个直接拒绝。这时由于没有配置nodelay，所以缓存区中的4个请求会按照rate的速率慢慢执行。

3.添加nodelay参数，通过设置burst可以允许nginx进行一定程度的并发，多余的请求会进入缓存区，慢慢处理，起到了对抗一些突发流量的目的，但是如果队列设置比较大，那么根据rate速率执行的时间比较长，就会导致用户等待时间加大。这里可以通过配置nodelay参数，表示缓存区中的请求立即执行，无需等待rate速率。

​ 借用示例2中的例子，假设某个IP在10ms内发送了6个请求，这时第一个请求被立即执行，第25个放入缓存区，第6个直接拒绝。这时由于配置了nodelay，第15基本上可以说是同时执行。所以这种情况下，系统的瞬时QPS会超过rate设置的阈值。

对比2和3：从例子的结果来看，示例2和示例3的请求处理的个数是一致的，只不过示例3的花费时间减少了。但是虽然burst和nodelay能够降低突发请求的处理事件，但是长期来看并不会提高吞吐量的上限。长期吞吐率是由rate决定，因为nodelay只能保证burst缓存中的请求被立即执行，但是nginx会限制队列元素的释放速度。

location匹配规则

  • =：表示精确匹配
  • ^~：表示uri以一个常规字符串开头
  • ~：表示区分大小写的匹配规则
  • ~*：表示不区分大小写
  • !~和!~*：表示区分大小写不匹配或者不区分大小写不匹配规则
  • /：通用匹配，任何请求都可以匹配到

匹配优先级顺序：首先匹配=,其次匹配^~,接着按顺序匹配正则表达式，最后最后匹配/,当有匹配成功则停止匹配。

假设nginx.conf中配置的顺序如下：

  1. http://localhost/→ 匹配A
  2. http://localhost/login/ → 匹配B
  3. http://localhost/register → 匹配H
  4. http://localhost/static/a.html → 匹配C
  5. http://localhost/a.png → 匹配规则D
  6. http://localhost/a.PNG → 匹配规则E
  7. http://localhost/a.xhtml → 匹配规则H
  8. http://localhost/category/id/1111 → 匹配规则H

也可以通过内置变量获取一些特定值进行路由控制。

root、alias和proxy_pass的区别：

  • root直接将location后匹配规则拼接到root后面进行访问。
  • alias别名，将location后匹配规则与alias替换。
  • proxy_pass转发到其他服务器上

负载均衡器

​ 在nginx中使用upstream模块设置负载均衡器。通过upstream设置两台服务器地址，在server中配置反向代理。默认情况下upstream采用轮询的方式。

​ nginx也能进行四层转发，使用的是stream，区别于upstream。可以实现端口到端口的转发。
